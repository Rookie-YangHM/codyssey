# 문제 5: 반달곰 커피의 홈페이지 관리의 시작

## 기술적 설명

상우는 Git을 사용할 수 있는 환경이 조성되자 이제까지 작업했던 내용들을 버전으로 관리해야겠다는 생각이 들었다.  
사실 아직은 별 내용이 없지만 그래도 여기서부터 시작해서 버전도 관리하고 발전시켜나가면 되겠다고 생각했다.

그리고 첫 번째 버전을 등록하려고 하니 뭔가 준비해야 할 작업들이 보이기 시작했다.  
상우는 Git에서 제공하는 명령을 하나하나 살펴보며 첫 번째 버전을 등록하기 시작했다.

---

## 수행 과제

### ✅ 1. **기초 Git 명령어 학습**

- 터미널에서 `git help` 명령어를 입력하고 출력되는 주요 명령어 목록을 확인한다.
- `git commit --help` 명령어를 통해 커밋 명령어의 다양한 사용법을 익힌다.

---

### ✅ 2. **정렬 프로그램 작성 - `sort_calculator.py`**

#### 📌 기능 요구사항

1. **입력 형식**
   - 터미널에서 `input()`을 사용하여 숫자들을 공백으로 구분하여 입력받는다.

2. **출력 형식**
   - 정렬된 숫자들을 오름차순으로 다음과 같은 형식으로 출력한다:
     ```
     Sorted: <숫자1> <숫자2> <숫자3> ...
     ```
   - 출력되는 숫자들은 반드시 소수점을 포함한 실수 (`float`) 형태로 출력한다.

3. **예외 처리**
   - 입력에 숫자가 아닌 값이 포함되었거나 입력이 비어 있는 경우 `"Invalid input."` 을 출력한다.

#### 🧱 구현 방식 및 기술 요구사항

- Python 내장 함수인 `sorted()` 와 리스트 메서드인 `.sort()` 사용을 금지한다. (자동 채점에서 확인됨)
- 반드시 직접 정렬 알고리즘을 구현해야 한다. (버블 정렬, 선택 정렬 등 자유롭게 구현 가능)
- 입력값은 반드시 터미널에서 `input()`을 사용하고, 공백으로 나누어 처리 (`split()`) 한다.
- 숫자 변환은 반드시 `float()`로 처리한다.
- 프로그램은 다음 형태로 작성하여 실행된다:

```python
if __name__ == "__main__":
    main()
```

---

### ✅ 3. **Git 저장소 초기화 및 커밋**

- 터미널에서 `git help`를 입력해서 출력되는 주요 명령어들 중 아래 사용할 명령어를 선택한다.
- 터미널에서 `git commit --help`를 입력하고 웹 브라우저로 뜨는 매뉴얼 페이지를 참조해서 사용 방법을 익힌다.
- Git의 스테이징 영역에 `sort_calculator.py`를 추가한다.
- Git 저장소에 `sort_calculator.py`를 커밋한다. 메시지는 Visual Studio Code에서 `first commit`으로 저장한다.
- 각각의 Git 명령어 사용 시 `git status` 등으로 상태를 확인한다.

---

## 개발환경

- Git에서 기본 제공되는 명령어를 터미널에서 실행하며, 별도의 도구를 사용하지 않는다.
- 커밋 명령어 사용 시 인자값 없이 Visual Studio Code에서 메시지를 저장한다.

## 제약조건

- 커밋 명령어 사용 시 인자값 없이 Visual Studio Code에서 메시지를 저장한다.
